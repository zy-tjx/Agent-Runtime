## 什么是 RAG（检索增强生成）？
AG 是一种将信息检索与文本生成相结合的革命性技术框架，其核心目的是利用外部知识库来弥补大型语言模型（LLM）的固有缺陷，如知识滞后和事实性“幻觉”。它的基本逻辑是“先查资料再回答”。RAG 的核心架构遵循“检索-增强-生成”的流程：首先在离线阶段将文档切分并转化为向量存入数据库；在线收到用户查询后，系统会检索出最相关的上下文片段，并将其与原始问题拼接，作为增强后的提示词输入给 LLM，最终生成准确、基于事实的回答。

## 为什么需要 RAG 技术，而不是直接微调大模型？
虽然微调可以提升模型的领域特定语气，但在注入大规模、快速变化的外部知识时，RAG 具有显著优势。首先，RAG 无需昂贵的训练成本，知识更新只需修改数据库而非重新训练模型。其次，RAG 具有极高的可解释性，因为生成答案所引用的具体文档片段是可以追溯的。最后，RAG 能有效防止模型产生幻觉，因为它强制模型基于提供的真实上下文进行回答，而不是依赖其内部过时的记忆。

## RAG 系统的核心架构包含哪三个关键阶段？
RAG 的工作流程主要分为三个阶段：首先是索引阶段（离线），将非结构化文档分割成小块（Chunks），通过嵌入模型转化为向量，并存入专门的向量数据库；其次是检索阶段（在线），将用户的自然语言查询同样转化为向量，在数据库中进行相似性搜索，召回最相关的 Top-K 个文档块；最后是生成阶段（在线），将检索到的上下文与原始查询组装成一个完整的 Prompt，发送给 LLM，由 LLM 结合这些信息生成最终的答案。

## RAG 系统是如何从基础架构向高级架构演进的？
RAG 的演进主要经历了三个层次：最基础的是 Naive RAG，它采用简单的“查询-检索-拼接-生成”流程，容易受到噪声干扰。随后发展为 Advanced RAG，它在检索前后引入了优化策略，如检索前的查询改写和检索后的重排序（Reranking），以提升上下文的精确度。目前最先进的是 Modular RAG，它将系统解耦为多个可插拔的模块（如搜索模块、记忆模块、融合模块），支持更复杂的多源检索和编排，大大增强了处理复杂任务和长程依赖的能力。

## 构建高效的 RAG 系统需要关注哪些关键技术组件？
关键技术组件主要包括四个方面：首先是文本分块策略，需要根据文档结构合理设定切分长度和重叠度，确保块的语义完整性；其次是高质量的嵌入模型，它将文本映射到语义空间，决定了检索的准确度；第三是向量数据库，它支持高效的近似最近邻搜索和高维向量存储；最后是混合检索与重排序技术，结合稀疏检索（如 BM25）和密集检索（向量搜索）的优势，并通过重排序模型对候选结果进行精排，显著提升召回率和精确率。

## 在生产环境中，有哪些策略可以优化 RAG 的检索效果？
生产环境中常用的优化策略有：查询改写，即在检索前对用户的原始问题进行扩展或重写，以提高召回率；上下文压缩，当检索内容过多时，通过摘要或过滤技术剔除无关信息，只保留核心事实并适配 LLM 的上下文窗口；以及多查询融合，将用户的复杂问题拆解为多个子查询并行检索，最后综合答案，这对于处理多跳推理问题特别有效。

## 常见问题
**Q: 如何评估一个 RAG 系统的好坏？主要关注哪些核心指标？**
**A**:评估 RAG 系统通常采用多维度的指标。在检索层面，关注精确率（Precision，返回的结果是否相关）和召回率（Recall，是否找全了相关信息）。在生成层面，除了传统的 BLEU 和 ROUGE，更关注 RAG 特有的指标，如上下文精确度（Context Precision，评估检索到的上下文对答案的支持程度）和答案相关性（Answer Relevance，评估生成的答案是否切题且无幻觉）。

**Q: RAG 系统如何应对企业内部知识库的频繁更新？**
**A**:应对数据新鲜度挑战主要依赖于构建自动化的数据管道。通常的做法是引入消息队列监听数据源的变更日志，一旦有文档新增、修改或删除，就触发 ETL 流程，自动更新向量数据库中的对应向量。为了保证服务不中断，还可以采用版本控制策略，允许新旧索引并存，并通过别名切换实现平滑过渡。

**Q: 为什么有时候 RAG 检索到了正确的文档，但 LLM 还是给出了错误答案？**
**A**:这种现象被称为“上下文忽略”或“中间丢失”。LLM 在处理长文本时，往往对提示词的开头和结尾部分关注度较高，而忽略中间的内容。为了解决这个问题，一方面可以通过重排序技术尽量将最关键的上下文移到片段的前面；另一方面可以在 Prompt 设计中增加强约束，明确指示模型必须基于提供的上下文回答，并在输出后加入答案一致性校验或后处理过滤步骤。

**Q: 随着大模型上下文窗口越来越大（如支持128K甚至1M tokens），RAG 还有存在的必要吗？**
**A**:即使上下文窗口扩大，RAG 依然具有重要价值。首先，全量塞入长文档成本极高且效率低下，RAG 的检索筛选机制能显著降低 LLM 的输入长度和推理成本。其次，RAG 的核心优势在于知识的动态更新，无需调用昂贵的微调即可接入最新信息。最后，在超大规模知识库场景下，精准定位信息的能力远比简单的“大海捞针”式全量输入更具实用性和经济性。
